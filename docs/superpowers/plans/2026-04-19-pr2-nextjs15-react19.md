### Task 2: Fix next.config.js

**Files:**
- Modify: `next.config.js`

- [ ] **Step 1: Remove the rewrites() block**

The current `next.config.js` contains:

```js
/** @type {import('next').NextConfig} */
const nextConfig = {
  reactStrictMode: true,
  output: 'export',
  images: {
    unoptimized: true,
  },
  // Make sure the _redirects file is copied to the output directory
  // This is needed for Netlify redirects to work
  async rewrites() {
    return [];
  }
}

module.exports = nextConfig
```

Replace it entirely with:

```js
/** @type {import('next').NextConfig} */
const nextConfig = {
  reactStrictMode: true,
  output: 'export',
  images: {
    unoptimized: true,
  },
}

module.exports = nextConfig
```

The comment about `_redirects` is removed because it was incorrect — `_redirects` is already in `public/` and gets copied by Next.js static export automatically; the `rewrites()` function had nothing to do with it. The `rewrites()` block is also incompatible with `output: 'export'` in Next.js 15.

- [ ] **Step 2: Commit**

```bash
git add next.config.js
git commit -m "fix: remove rewrites() incompatible with output:export in Next 15"
```

### Task 3: Verify the build

**Files:** none (verification only)

- [ ] **Step 1: Run the build**

```bash
npm run build
```

Expected output includes:
```
✓ Compiled successfully
✓ Generating static pages (N/N)
Export successful
RSS feed generated successfully
```

If you see TypeScript errors about React types, the most likely cause is a component using the old `React.FC` type or `children: ReactNode` prop pattern. In this project's components (`src/components/layout.tsx`, `src/components/date.tsx`), check if they use explicit `children` props. React 19 removed the implicit `children` from `React.FC` — but this project's components don't use `React.FC`, so this should not be an issue.

If build fails with an error about `rewrites`, double-check Task 2 was completed correctly.
